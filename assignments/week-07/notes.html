  • call() method can be used to set the value of this inside a function to an object that is provided as the frst argument
  • apply() method can works in the same way as the call() method except the arguments of the funciton are provided as an array, even if there is only one argument.
  • A description property can be added to describe what a function does
  • A cache can save the result and return the item later
  • IFFE = Immediately Invoked Function Expression
  • Temporary Variables should be put inside IFFE
  • It is best to put the 'use strict' mode inside a function
  • Self contained code blocks can use the same variable name and not interfere with the code
  • A function can be defined and called as a variable
  • Init-time branching is to create a function that can rewrite itself
  • Recursive function is one that invokes itself until a certain condition is met
  • Callbacks can be used to facilitate event-driven asynchronous programming
  • Callback Hell is a term used to refer to a tangled mess of code when more than one callback is used in the same function
  • A promice represents the future result of an asynchronous operation
  • When a promise is created, it calls an asynchronous operation and is then said to be pending and remains in this state while the operation is taking place
  • Resolved - the asynchronous operation was completed successfully
  • Rejected - the asynchronous operation didn't work as expected, wasn't successfully completed or resulted in an error
  • Multiple promises can be chained together
  • Async Functions allow you to write asynchronous code as if it was synchronous
  • Generalized functions
  • Functions can return functions
  • A closure is a reference to a variable that was created inside the scope of another function, but is then kept alive and used in another part of the program
  • Closures not only have access to variables declared in a parent function's scope, they can aslo change the value of these variables
  • Generators are special functions used to produce iterators that maintain the state of a value
  • Pure Functions
    • The return value of a pure function should only depend on the values provided as arguments. It doesn't rely on values from somewhere else in the program
    • There are no side-effects. a pure function doesn't change any values or data elsewhere in the program. It only makes non-destructive data transformations and returns new values, rather than altering any of the underlying data
    • Referential transparency. Given the same arguments, a pure function will always return the same result
    • Any pure function must have:
      • At least one argument; otherwise the return value must depend on something other than the arguments of the function, breaking the first rule
      • A return value; otherwise there's no point in the function (unless it has changed something else in the program - in which case, it's broken the "no side-effects" rule
  • High-order functions are functions that accept another function as an argument
  • Currying is a process that involves the partial application of functions

    Chapter 13

  • Ajax is a technique that allows web pages to communicate asynchronously with a server, and it dynamically updates web pages without reloading
  • It is unusual for Ajax not to be used when a partial web page update is required
  • Fetch API is currently a living standard for requesting and sending data asynchronously across a network
  • Response Interface deals with the object that's returned when the promise is fulfilled
  • Response objects are: headers, url, redirected, type
  • redirect() method can be used to redirect to another URL
  • Text responses text() method takes a stream of text from the response, reads it to completion and then returns a promise that resolves to a USVSting objec that can be treated as a string in JavaScript
  • blob() method is used to read a file of raw data, such as an image or a spreadsheet
  • JSON is probably the mose common format for AJAX responses
  • Request objects are created using the Request() constructor and include the following properties: utl, method, headers, mode, cache, credentials, redirect
  • HTTP headers are used to pass on any additional information about a request or response
