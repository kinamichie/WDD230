Notes For Week 5

    Chapter 10

  • Errors
    • System error- there's a problem with the system or external devices with which the program is interacting
    • Programmer error- the program contains incorrect sytax or faulty logic; it could even be as simple as a typo
    • User error- the user has entered data incorrectly, which the program is unable to handle
  • An exception is an error that produces a return value that can then be used by the program to deal with the error
  • An exception will also produce a stack tace which is a sequence of functions or method calls that lead to the point where the error occurred
  • Warnings can occur if there's an error in the code that isn't enough to cause the program to crash. The program will still run but might not run properly
  • It's important to test your code and deal with errors
  • Strict mode produces more restrictions and warnings. Activate it by putting 'use strict' at the beginning of your js code or at the beginning of a function
  • Linting tools such as JS Lint, JS Hint, and ES Lint can be used to test the quality of JS code
  • Always look at browser support when determining a feature to use
  • The most basic form of debugging is to use the alert() method
  • Console.log can be used to show what is happening when the program is used to help find errors
  • Using the debugging tool in the browser is a great way to find errors
  • An error object can be created by the host environment when an exception occurs, or it can be created in the code const error = new Error(); const error = new Error('oops');
    • EvalError was used to identify errors when using the global eval() function
    • RangeError is thrown when a number is outside an allowable range of values
    • ReferenceError is thrown when there's an error in the code's syntax
    • Syntax Error is thrown when there's an error in the code's syntax
    • TypeError is thrown when there's an error in the type of value used; for example, a string is used when a number is expected
    • URIError is thrown when there's a problem incoding or decoding the URI
    • InternalError is a non-standard error that is thrown when an error occurs in the JavaScript engine. A common cause of this is too much recursion
    • Properties that are generally safe to use
      • The name property returns the name of the error constructor function used as a string, such as "Error" or "ReferenceError"
      • The message property returns a description of the error and should be provided as an argument to the Error constructor function
      • The stack property will return a stack trace for that error. This is a non-standard property and it's recommended that it is not safe to use in production sites
  • It is possible to throw your own exception with a throw statement
  • When an exception occurs, the program terminates with an error message
  • Use try, catch, and finally to handle exceptions (like Java)
  • Test-driven Development(TDD)
    • Write tests (that initially fail)
    • Write code to pass the tests
    • Refactor the code
    • Test refactored code
    • Write more tests for new features
  • Testing frameworks; Jest
  • Number cruncher can be used to test math code
