  • CSS3 transform property lets yo translate, rotate, scale, and/or skew any element on the page
  • Translation functions allow you to move elements left, right, up or down
  • use translatex or translatey functions coma move an element vertically or horizontally
  • The scale(x,y) functionscales an element by the defined factors horizontally then vertically transform: scale(1.5, 0.25);
  • The rotate() function rotates an element around the point of origin by a specified angle value
  • The skew(x,y) function specifies a skew along the x and y axes
  • You can control the origin from which your tansfors are applied
  • The order of transform function matter
  • CSS transitions are declared along with the regular styles on an element
    • Declare the original state of the element in the default style declaration
    • Declare the final state of your transitioned element; for example, a :hover state
    • Include the transition functions in your default style declaration using the transition properties, including: transition-property, transition-duration, transition-timing-function, and transition-delay
  • The transition-property defines the CSS properties of the element that should be transitioned, with all for all properties being the default
  • Its important to include a pre-state and a post-state
  • The transition-duration property sets how long the transition will take: the duration of time it takes to go from the default state to the transitioned state
  • The transition-timing-function lets you control the pace of the transition in even more granular detail. most common timing function are ease, linear, ease-in, ease-out, or ease-in-out
  • The transition-delay property makes it possble to introduce a delay before the transition begins
  • Functions can be combined into a shorthand declaration
  • Multiple transitions can happen in one call
  • A e is a snapshot that defines a starting or end point of any smooth transition
  • Keyframes can be specified in any order
  • Animation properties need two declarations for each property
  • animation-name is used to attach an animation to an element
  • animation-duration property defines the length of time an animation takes to complete one iteration
  • animation-timing-function determmines how the animation will progress over its duration
  • The animation-iteration-count property lets you define how many times the animation will play through
  • animation-direction property can change the default value for animation iterates
  • animation-delay property is used to define how many milliseconds or seconds to wait before the browser begins the animation
  • animation-fill-mode definew what happens before the first animation iteration begins and after the last animation iteration concludes
  • animation-play-state property defines whether the animation is running or paused
  • There is a shorthand property for all the animation properties
