    Chapter 9

  • A regular expression is a type of object and can be contruced with the RegExp() or forward slash (/) characters. /abc?;
  • Expression can be tested for matches console.log(/abc/.test("abcde")); //->true console.log(/avc/.test("abxde")); //->false
  • A hyphen between two characters can be used to indicate a range of characters, where the ordering is determined by the character's Unicode number
  • Built-in shortcut
    • \d Any digit character
    • \w An alphanumeric character ("word character")
    • \s any whitespace character)space, tab, newline, and similar
    • \D A character that is not a digit
    • \W a nonalphanumeric character
    • \S A nonwhitespace character
    • . Any character except for newline
  • A plus sign after something in a regular expression indicates that the element may be repeated more than once
  • A * sign has a similar meaning but also allows the pattern to match zero times
  • A question mark makes a part of a pattern otional, meaning it may occur zero times or one time
  • T indicate that a pattern should occur a precise number of times, used braces {} after an element
  • You can group sub-expressions but parenthesis must be used inbewteen
  • The exec method will return null if no match was found and return an object with information about the match otherwise
  • new Date() can be used to create and object for a specific time or date
